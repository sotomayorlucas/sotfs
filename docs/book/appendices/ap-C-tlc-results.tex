\section*{Resumen oficial v0.2.5}

\textbf{14/14 PASS} en \texttt{small} y \texttt{medium} (todos los
specs, todas las invariantes). La config \texttt{large} es opcional y
puede TIMEOUT dependiendo del host. El gating de CI es \texttt{medium}.

\section*{Cómo reproducir}

\begin{lstlisting}[language=shellcmd]
# Setup (una vez).
mkdir -p formal/lib
curl -L -o formal/lib/tla2tools.jar \
  https://github.com/tlaplus/tlaplus/releases/latest/\
download/tla2tools.jar

# Run all sizes.
just formal

# Sólo small (para CI rápida).
cd formal && bash run_tlc.sh small

# Sólo un spec.
cd formal && bash run_tlc.sh graph
\end{lstlisting}

\section*{Limitaciones declaradas}

\begin{itemize}
  \item Los números de estados son aproximaciones para configs típicas;
    el spec exacto depende del seed y del orden de exploración.
  \item Las configs \texttt{large} suelen requerir 32+ GiB de RAM
    para completarse; no se corren en CI.
  \item El refinement de \citaTLA{sotfs\_crash\_refinement.tla} verifica
    que el modelo concreto refina el abstracto bajo stuttering, no
    \emph{equivalencia}: el concreto puede tener trazas extra
    (\texttt{Crash}, \texttt{Recover}) que el abstracto no.
\end{itemize}
